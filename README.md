# The Archive вЂ“ A Blog by GenialSmileAdam

The Archive is a feature-rich blog platform built with Flask. It allows users to register, log in, read posts, and leave comments. The site administrator (the user with id = 1) can create, edit, and delete posts, and also access JSON API endpoints for posts and users. The project integrates FlaskвЂ‘SQLAlchemy, FlaskвЂ‘Login, CKEditor for rich text editing, Gravatar for user avatars, and a contact form that sends emails via SMTP.

---

## вњЁ Features

- **User authentication** вЂ“ Register, log in, log out, and edit your profile.
- **RoleвЂ‘based access** вЂ“ Only the admin (user id 1) can create, edit, or delete posts.
- **Blog posts** вЂ“ Rich text editing with CKEditor, automatic date stamp, and post image.
- **Comments** вЂ“ LoggedвЂ‘in users can comment on posts; comments are sanitized to prevent XSS.
- **Contact form** вЂ“ Sends a message to the site owner via Gmail SMTP.
- **Gravatar integration** вЂ“ User avatars are displayed next to comments.
- **RESTful API endpoints** вЂ“ Admin can retrieve all posts or all users by providing a password.
- **CSRF protection** вЂ“ Enabled via FlaskвЂ‘WTF.
- **Responsive design** вЂ“ Based on the Clean Blog Bootstrap theme.

## рџљЂ Getting Started

Follow these instructions to run the project locally.

### Prerequisites

- Python 3.11 or higher
- pip (Python package installer)
- Git
- (Optional) A Gmail account for the contact form

### Installation

1. **Clone the repository**

   ```bash
   git clone https://github.com/GenialSmileAdam/the-archive.git
   cd the-archive
   ```

2. **Create and activate a virtual environment**

   ```bash
   python -m venv venv
   source venv/bin/activate      # On Windows: venv\Scripts\activate
   ```

3. **Install dependencies**

   ```bash
   pip install -r requirements.txt
   ```

4. **Set up environment variables**

   Create a `.env` file in the project root and add the following:

   ```env
   SECRET_KEY=your-secret-key-here
   DATABASE_URI=sqlite:///posts.db          # or your PostgreSQL URI
   MY_EMAIL=your-email@gmail.com
   MY_PASSWORD=your-app-password             # Use an app password for Gmail
   ```

   - `SECRET_KEY`: A random string (e.g., generated by `secrets.token_hex(16)`).
   - `DATABASE_URI`: Defaults to SQLite if not set. For PostgreSQL use `postgresql://username:password@localhost/dbname`.
   - `MY_EMAIL` / `MY_PASSWORD`: Used by the contact form. For Gmail, enable 2вЂ‘Factor Authentication and create an [App Password](https://myaccount.google.com/apppasswords).

5. **Create the database tables**

   The tables are defined using SQLAlchemy. You can create them by running a oneвЂ‘time script:

   ```bash
   flask shell
   >>> from app import app, db
   >>> with app.app_context():
   ...     db.create_all()
   >>> exit()
   ```

   Alternatively, uncomment the lines in `app.py`:

   ```python
   # with app.app_context():
   #     db.create_all()
   ```

   and run `python app.py` once.

6. **Run the application**

   ```bash
   python app.py
   ```

   The server will start at `http://127.0.0.1:5001`.

7. **Access the blog**

   Open your browser and go to `http://127.0.0.1:5001`.  
   Register a new user вЂ“ the very first registered user will automatically become the admin (id = 1).

## рџ”Њ API Endpoints

Two readвЂ‘only JSON endpoints are available for the admin. They require a password query parameter that matches the admin userвЂ™s password (the user with id = 1).

| Endpoint               | Method | Description                          | Example                                   |
|------------------------|--------|--------------------------------------|-------------------------------------------|
| `/get-all-posts`       | GET    | Returns all blog posts               | `/get-all-posts?password=admin_password` |
| `/get-all-users`       | GET    | Returns all registered users         | `/get-all-users?password=admin_password` |

**Response format**:

```json
{
  "num_of_posts": 5,
  "posts": [ { "id": 1, "title": "...", ... } ]
}
```
